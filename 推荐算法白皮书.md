# 1. 召回（Recall / Candidate Generation）

召回是推荐系统的第一阶段，从海量物品库（百万~亿级）中快速筛选出用户可能感兴趣的候选集（几千量级）。召回阶段的核心要求是**高效**和**高覆盖**，可以牺牲一定精度。通常采用多路召回策略，每条召回通道独立运行，结果取并集后送入排序阶段。

---

## 1.1 协同过滤（Collaborative Filtering）

> 核心思想：基于用户行为数据（点击、收藏、点赞等），利用"用户-物品"交互矩阵中的共现关系进行推荐，不依赖物品本身的内容或属性。

### 1.1.1 ItemCF：基于物品的协同过滤

**原理**

如果用户喜欢物品 $i_1$，而物品 $i_1$ 和物品 $i_2$ 相似（受众重合度高），那么推荐 $i_2$ 给用户。

**相似度计算**

- 喜欢物品 $i_1$ 的用户集合 $W_1$，喜欢物品 $i_2$ 的用户集合 $W_2$，交集 $V = W_1 \cap W_2$。
- 基础版（Jaccard 变体）：

$$
\text{sim}(i_1, i_2) = \frac{|V|}{\sqrt{|W_1| \cdot |W_2|}}
$$

- 考虑喜欢程度的版本（余弦相似度）：

$$
\text{sim}(i_1, i_2) = \frac{\sum_{v \in V} \text{like}(v, i_1) \cdot \text{like}(v, i_2)}{\sqrt{\sum_{u_1 \in W_1} \text{like}^2(u_1, i_1)} \cdot \sqrt{\sum_{u_2 \in W_2} \text{like}^2(u_2, i_2)}}
$$

**线上召回流程**

| 步骤 | 操作 | 描述 |
|------|------|------|
| 离线 | 用户→物品索引 | 记录用户最近交互过的物品 ID（last-n） |
| 离线 | 物品→物品索引 | 计算物品两两相似度，每个物品索引 top-k 相似物品 |
| 线上 | 召回 | 用户 → last-n 物品 → 每个物品的 top-k 相似物品 |
| 线上 | 打分 | $\text{score}(user, item) = \sum_j \text{like}(user, item_j) \times \text{sim}(item_j, item)$ |

**优缺点**

| 优点 | 缺点 |
|------|------|
| 实现简单，可解释性强 | 只基于行为数据，未利用物品属性 |
| 计算离线化，线上速度快 | 冷启动问题：新物品无交互数据 |
| 适合物品更新不频繁的场景 | 热门物品容易与所有物品相似（需降权） |

---

### 1.1.2 UserCF：基于用户的协同过滤

**原理**

如果用户 $u_1$ 和用户 $u_2$ 兴趣相似，而 $u_2$ 喜欢某物品，则推荐该物品给 $u_1$。

$$
\text{score}(user, item) = \sum_j \text{sim}(user, user_j) \times \text{like}(user_j, item)
$$

**用户相似度计算**

- 用户 $u_1$ 喜欢的物品集 $J_1$，用户 $u_2$ 喜欢的物品集 $J_2$，交集 $I = J_1 \cap J_2$。
- 基础版：

$$
\text{sim}(u_1, u_2) = \frac{|I|}{\sqrt{|J_1| \cdot |J_2|}}
$$

- 降低热门物品权重（IIF 思想）：热门物品人人都喜欢，无法体现个性化。

$$
\text{sim}(u_1, u_2) = \frac{\sum_{l \in I} \frac{1}{\log(1 + n_l)}}{\sqrt{|J_1| \cdot |J_2|}}
$$

其中 $n_l$ 是喜欢物品 $l$ 的用户数量。

**线上召回流程**

1. 用户 → top-k 相似用户（用户→用户索引）
2. top-k 用户 → 各自的 last-n 物品（用户→物品索引）
3. 汇总打分排序

**优缺点**

| 优点 | 缺点 |
|------|------|
| 能发现新兴趣（社交发现） | 用户量级大，相似度计算开销大 |
| 适合用户兴趣多样的场景 | 新用户冷启动问题 |
| 推荐结果多样性好 | 不如 ItemCF 稳定 |

**ItemCF vs UserCF**

| 维度 | ItemCF | UserCF |
|------|--------|--------|
| 适用场景 | 物品少、用户多（电商） | 用户少、物品多（新闻、社交） |
| 个性化 | 基于用户自身历史 | 基于相似群体 |
| 实时性 | 物品相似度离线算好，较稳定 | 用户兴趣变化快，需频繁更新 |

### 1.1.3 Swing 模型

**原理**

针对 ItemCF 中的"小圈子问题"（如微信群内互相转发导致虚假相似）和"泛用户问题"（大量随意浏览的用户带来的噪声），对用户对设置权重。

$$
\text{sim}(i_1, i_2) = \sum_{u_1 \in V} \sum_{u_2 \in V} \frac{1}{\alpha + \text{overlap}(u_1, u_2)}
$$

- $V$ 是同时喜欢 $i_1$ 和 $i_2$ 的用户集合
- $\text{overlap}(u_1, u_2)$ 是两个用户共同交互的物品数量
- 两用户交互重叠越大，说明可能在"小圈子"中，权重越低

**优缺点**

| 优点 | 缺点 |
|------|------|
| 有效缓解小圈子问题 | 计算复杂度比 ItemCF 高 |
| 对噪声用户有天然抵抗力 | 超参数 α 需要调优 |
| 工业界广泛使用（阿里等） | |

## 1.2 矩阵补充（Matrix Completion）

**原理**

将用户 ID 和物品 ID 通过 Embedding 层映射为向量，用向量内积预测用户对物品的兴趣分数。

- 数据集：$(u, i, y)$，其中 $y$ 是兴趣分数（0-4 分，涵盖曝光无点击到转发）
- 优化目标：

$$
\min_{A, B} \sum_{(u, i, y) \in \Omega} (y - \langle a_u, b_i \rangle)^2
$$

**优缺点**

| 优点 | 缺点 |
|------|------|
| 思路简单，易于理解 | 只用了 ID Embedding，未利用物品/用户属性 |
| | 负样本选取方式不合理（应区分未曝光和曝光未点击） |
| | 内积不如余弦相似度（未归一化） |
| | 回归损失不如交叉熵 |

> 矩阵补充可以看作双塔模型的简化版本，实际效果不如双塔模型。

---

## 1.3 双塔模型（Two-Tower Model）

**原理**

分别用两个独立的神经网络（塔）编码用户特征和物品特征，输出各自的表征向量，用余弦相似度衡量匹配程度。

- 用户塔输入：用户 ID、离散特征、连续特征
- 物品塔输入：物品 ID、离散特征、连续特征
- 两个塔各自输出一个向量 $a$（用户）和 $b$（物品），通过 $\cos(a, b)$ 计算相似度

**训练方式**

| 方式 | 描述 | 损失函数 |
|------|------|----------|
| Pointwise | 每个样本独立二分类 | 交叉熵 |
| Pairwise | 一正一负成对训练 | Triplet Hinge Loss: $L = \max\{0, \cos(a,b^-) + m - \cos(a,b^+)\}$ |
| Listwise | 一正多负列表训练 | Softmax + 交叉熵 |

**正负样本选择**

| 类型 | 策略 |
|------|------|
| 正样本 | 曝光且点击的物品；过采样冷门物品，降采样热门物品 |
| 简单负样本 | 未被召回的物品；非均匀采样，概率 ∝ 点击次数^0.75 |
| Batch 内负样本 | 同 batch 内其他用户的正样本作为当前用户的负样本 |
| 困难负样本 | 被召回但被粗排/精排淘汰的物品 |

**上线部署**

| 阶段 | 操作 |
|------|------|
| 离线 | 用物品塔预计算所有物品 embedding，建立 ANN 索引（如 FAISS） |
| 线上 | 用户发起请求 → 用户塔实时计算用户向量 → 最近邻检索 |

> 用户兴趣动态变化，需要在线推理；物品特征相对稳定，可以缓存。

**模型更新**

| 方式 | 描述 |
|------|------|
| 全量更新 | 每天凌晨用前一天数据训练 1 epoch（基于昨日模型参数，非随机初始化） |
| 增量更新 | Online Learning，实时收集数据，流式更新 ID Embedding 参数 |

**优缺点**

| 优点 | 缺点 |
|------|------|
| 利用丰富的用户/物品特征 | 用户和物品特征独立编码，无法建模交叉特征 |
| 物品 embedding 可离线计算，线上仅需推理用户塔 | 模型表达能力受限于向量内积/余弦相似度 |
| 支持 ANN 高效检索 | 负样本选择策略对效果影响很大 |
| 工业界最主流的召回模型 | |

## 1.4 Deep Retrieval

**原理**

不同于双塔模型用向量检索，Deep Retrieval 用"路径"来表示物品，通过神经网络预测用户对路径的兴趣。

- 物品表示：每个物品映射到若干条路径，一条路径由多层节点组成，如 path = [a, b, c]
- 预测模型：逐层预测用户对节点的兴趣

$$
p(a, b, c | x) = p_1(a | x) \times p_2(b | a; x) \times p_3(c | a, b; x)
$$

**线上召回**

1. 给定用户特征 $x$，用 Beam Search 在路径树上搜索分数最高的 $s$ 条路径
2. 通过路径→物品索引，找到对应物品

**训练**

| 阶段 | 目标 |
|------|------|
| 学习神经网络参数 | 如果用户点击了物品，更新网络使对应路径分数变大 |
| 学习物品表征 | 根据用户兴趣分数确定物品与路径的对应关系，同时控制每条路径上的物品数量 |

**优缺点**

| 优点 | 缺点 |
|------|------|
| 不依赖 ANN 索引，路径搜索效率高 | 模型结构复杂，训练难度大 |
| 物品路径可以灵活更新 | 路径层数和宽度需要调优 |
| 适合超大规模物品库 | Beam Search 可能遗漏最优路径 |

## 1.5 其他召回通道

| 召回通道 | 索引结构 | 是否个性化 | 特点 |
|----------|----------|-----------|------|
| GeoHash 地理位置召回 | GeoHash → 优质笔记（按时间倒排） | ❌ | 推荐附近内容 |
| 同城召回 | 城市 → 优质笔记 | ❌ | 粒度更粗的地理召回 |
| 关注作者召回 | 用户→关注作者→最新笔记 | ✅ | 社交关系驱动 |
| 有交互作者召回 | 用户→有交互作者→最新笔记 | ✅ | 隐式兴趣 |
| 相似作者召回 | 用户→感兴趣作者→相似作者→最新笔记 | ✅ | 扩展发现 |
| 缓存召回 | 精排高分但未被重排选中的笔记 | ✅ | 避免浪费精排结果 |

**曝光过滤：Bloom Filter**

- 用于判断物品是否已对用户曝光过，避免重复推荐
- k 个哈希函数，m bits 哈希表，n 个物品
- 误伤概率：

$$
\left(1 - \exp\left(-\frac{kn}{m}\right)\right)^k
$$

- 判断为 No → 一定不在集合中；判断为 Yes → 可能在集合中（存在误判）

# 2. 粗排（Pre-Ranking）

粗排是召回和精排之间的中间环节。召回输出几千个候选物品，精排模型复杂但计算开销大，无法处理全部候选。粗排用较轻量的模型从中筛选出几百个候选送入精排。

## 2.1 三塔模型

**架构**

| 塔 | 输入特征 | 特点 |
|----|----------|------|
| 用户塔（较大） | 用户特征 + 场景特征 | 每个请求推理 1 次 |
| 物品塔（较大） | 物品特征（相对静态） | 可缓存输出向量，避免绝大部分推理 |
| 交叉塔（较小） | 统计特征 + 交叉特征 | 每个候选物品推理 1 次 |

**推理流程**

1. 用户塔：给定 1 个用户 → 推理 1 次
2. 物品塔：n 个召回物品 → 先查缓存，未命中才推理
3. 交叉塔：n 个物品 → n 次推理
4. 三个塔的输出融合，送入上层网络，给 n 个物品打分

**优缺点**

| 优点 | 缺点 |
|------|------|
| 物品塔可缓存，大幅降低计算量 | 三塔独立编码，交叉能力弱于精排 |
| 比精排轻量，比双塔（召回）更精确 | 交叉塔仍需 n 次推理 |
| 架构灵活，可以调整各塔大小 | |

# 3. 精排（Ranking）

精排是推荐系统中模型最复杂、特征最丰富的阶段，对粗排输出的几百个候选进行精细打分和排序。

## 3.1 多目标模型（Multi-Task Model）

**原理**

对每个候选物品，同时预测多个业务指标的概率（点击率、点赞率、收藏率、转发率等），将预估分数融合后排序。

**输入特征**

| 类型 | 示例 |
|------|------|
| 用户特征 | 用户 ID、性别、年龄、兴趣标签 |
| 物品特征 | 物品 ID、类目、关键词、发布时间 |
| 统计特征 | 用户/物品近 30 天曝光数、点击数、点赞数 |
| 场景特征 | 请求时间、地点、设备信息 |

**模型结构**

特征拼接 → 共享底层网络（Shared Bottom） → 输出向量 → 多个任务头（各自 FC + Sigmoid）→ 输出各业务指标概率

**训练**

- 每个任务头是二分类（是否点击/点赞/收藏/转发）
- 损失函数：加权交叉熵之和

$$
\text{Loss} = \sum_{i=1}^{4} \alpha_i \cdot \text{CrossEntropy}(y_i, p_i)
$$

- 类别不平衡处理：负样本降采样，然后用校准公式修正预估值

**优缺点**

| 优点 | 缺点 |
|------|------|
| 多任务共享底层，节省计算 | 任务间可能存在冲突（跷跷板效应） |
| 统一建模，方便维护 | 共享底层可能限制各任务的学习 |

---

## 3.2 MMoE：Multi-gate Mixture-of-Experts

**原理**

改进 Shared Bottom 结构，使用多个 Expert 网络（结构相同但参数不同），每个任务有独立的 Gate Network 来决定各 Expert 的贡献权重。

**结构**

1. 输入特征送入 m 个 Expert 网络，各输出一个向量
2. 每个任务有一个 Gate Network，输入相同特征，输出 m 个权重（Softmax）
3. 每个任务的输入 = Expert 输出的加权平均
4. 加权平均结果送入任务头进行预测

**极化问题（Polarization）**

- Softmax 输出可能接近 one-hot，导致只使用单个 Expert，浪费其他 Expert 的参数
- 解决方案：对 Softmax 输出做 Dropout（每个 Expert 有 10% 概率被丢弃）

**优缺点**

| 优点 | 缺点 |
|------|------|
| 各任务可以学到不同的 Expert 组合 | 可能出现极化问题 |
| 缓解任务间的冲突 | 比 Shared Bottom 参数量更大 |
| 工业界广泛使用 | Gate Network 设计需要调优 |

---

## 3.3 特征交叉模型

> 目的：自动学习特征之间的组合关系，提升模型表达能力。

### 3.3.1 FM：因式分解机

**原理**

在线性模型基础上加入二阶特征交叉项，并用低秩矩阵分解降低参数量。

- 线性模型：$p = b + \sum_{i=1}^{d} w_i x_i$，参数量 $O(d)$
- 加入交叉项：$p = b + \sum_i w_i x_i + \sum_{i=1}^{d} \sum_{j=i+1}^{d} u_{ij} x_i x_j$，参数量 $O(d^2)$
- FM 用低秩分解：$u_{ij} \approx \langle v_i, v_j \rangle$，参数量降至 $O(kd)$（k 为向量维度）

**优缺点**

| 优点 | 缺点 |
|------|------|
| 参数量从 $O(d^2)$ 降到 $O(kd)$ | 只能建模二阶交叉 |
| 稀疏数据下也能有效学习 | 高阶交叉需要扩展（如 DeepFM） |

### 3.3.2 DCN：Deep & Cross Network

**原理**

设计专门的 Cross Layer，显式建模高阶特征交叉。

- Cross Layer 公式：

$$
x_{i+1} = x_0 \odot (W \cdot x_i + b) + x_i
$$

- $x_0$ 是原始特征，$\odot$ 是逐元素相乘（Hadamard product）
- 每一层 Cross Layer 增加一阶交叉，L 层可以建模 L+1 阶交叉

**网络结构**

输入特征同时送入 Cross Network（多层 Cross Layer）和 Deep Network（普通 DNN），两部分输出拼接后做最终预测。

**优缺点**

| 优点 | 缺点 |
|------|------|
| 显式建模高阶交叉，可控性好 | Cross Layer 的表达能力有理论上界 |
| 参数高效（每层只有 W 和 b） | 实际效果取决于数据特点 |
| 与 DNN 互补 | |

### 3.3.3 LHUC / PPNet

**原理**

Learning Hidden Unit Contribution（LHUC），快手用于精排时称为 PPNet。

- 核心思想：用用户特征通过一个小型神经网络（多层 FC + 2×Sigmoid）生成"门控信号"
- 门控信号对主网络的隐藏层进行逐元素缩放，实现个性化的特征放大/抑制

**优缺点**

| 优点 | 缺点 |
|------|------|
| 提高个性化程度 | 增加了网络参数和推理时间 |
| 实现简单，即插即用 | 效果依赖于用户特征的质量 |

### 3.3.4 SENet

**原理**

对离散特征进行 field-wise（字段级别）加权，学习每个特征 field 的重要性。

**流程**

1. m 个特征，每个是 k 维向量 → $m \times k$ 矩阵
2. Average Pooling → $m \times 1$ 向量
3. FC + ReLU 降维到 $\frac{m}{r} \times 1$（r 为压缩比，通常 4 或 8）
4. FC + Sigmoid 升维到 $m \times 1$
5. 与原始矩阵逐行相乘（field-wise reweighting）

**优缺点**

| 优点 | 缺点 |
|------|------|
| 自动学习特征重要性 | 需要所有特征 embedding 维度一致 |
| 轻量级，计算开销小 | 只做 field-wise 加权，不做交叉 |
| 与其他交叉方法可叠加使用 | |

### 3.3.5 Bilinear Cross

**原理**

用可学习的矩阵 $W$ 建模特征之间的交互，比简单内积更有表达力。

| 方式 | 公式 | 输出 |
|------|------|------|
| Bilinear 内积 | $e_i^T W e_j$ | 标量 |
| Bilinear 哈达玛乘积 | $e_i \odot (W e_j)$ | 向量 |

**优缺点**

| 优点 | 缺点 |
|------|------|
| 比内积表达力更强 | 参数矩阵 $W$ 增加存储和计算 |
| 可以捕获非对称关系 | 特征组合数多时计算量大 |

## 3.4 预估分数融合

多目标模型输出多个预估分数后，需要融合为一个排序分数。常见方案：

| 方案 | 公式 | 适用场景 |
|------|------|----------|
| 简单加权和 | $p_{click} + w_1 \cdot p_{like} + w_2 \cdot p_{collect} + \ldots$ | 通用 |
| 点击率乘加权和 | $p_{click} \cdot (1 + w_1 \cdot p_{like} + w_2 \cdot p_{collect} + \ldots)$ | 点击是核心指标 |
| 幂次乘积（短视频） | $(1 + w_1 \cdot p_{time})^{\alpha_1} \cdot (1 + w_2 \cdot p_{like})^{\alpha_2} \cdot \ldots$ | 短视频 |
| 排名融合（快手） | $\sum_i \frac{1}{r_i^{\alpha_i} + \beta_i}$（r 为各指标排名） | 多指标均衡 |
| 漏斗概率（电商） | $p_{click}^{\alpha_1} \times p_{cart}^{\alpha_2} \times p_{pay}^{\alpha_3} \times price^{\alpha_4}$ | 电商转化漏斗 |

## 3.5 视频播放建模

### 播放时长

- 直接回归效果不好
- 解决方案：将播放时长离散化后用交叉熵优化

### 完播率

| 方法 | 描述 |
|------|------|
| 回归 | 完播率 $y \in [0,1]$，用交叉熵优化：$\text{loss} = y \cdot \log p + (1-y) \cdot \log(1-p)$ |
| 二分类 | 播放超过 80% 定义为完播，做 0/1 分类 |

> 注意：完播率不能直接用于融分，因为短视频天然完播率高于长视频。需要做归一化：$p_{finish} = \frac{\text{预估完播率}}{f(\text{视频时长})}$

## 3.6 排序模型特征工程

### 特征分类

| 类别 | 特征项 |
|------|--------|
| 用户画像 | 用户 ID、性别、年龄、新老用户、活跃度、兴趣类目/关键词/品牌 |
| 物品画像 | 物品 ID、发布时间、GeoHash/城市、标题/类目/关键词、内容质量（字数、图片数、美学分等） |
| 用户统计特征 | 近 30 天曝光/点击/点赞/收藏数（按图文/视频、按类目分桶统计） |
| 物品统计特征 | 近 30 天曝光/点击/点赞等（按用户性别、年龄分桶统计） |
| 场景特征 | 用户定位、当前时刻、是否节假日、手机品牌/型号/OS |

### 特征处理

| 特征类型 | 处理方式 |
|----------|----------|
| 离散特征（ID、类目、城市等） | Embedding |
| 连续特征（年龄、字数、时长等） | 分桶离散化 |
| 长尾连续特征（曝光量等） | $\log(1 + x)$ 平滑 |
