# 三塔模型

用户塔 + 物品塔 + 交叉塔

![Screenshot 2026-03-17 at 13.08.48.png](3%20%E7%B2%97%E6%8E%92%E6%A8%A1%E5%9E%8B/Screenshot_2026-03-17_at_13.08.48.png)

- 用户塔（很大）：
    - 用户特征
    - 场景特征
- 物品塔（较大）：
    - 物品特征：相对静态
    - 缓存物品塔的输出向量，避免绝大部分推理
- 交叉塔（较小）：
    - 统计特征
    - 交叉特征

## 推理流程

1. 一个用户 → 一个推理
2. n个召回的物品 → 首先检查物品塔的缓存，如果没有命中缓存，则需要做推理
3. 交叉塔 → n次推理
4. 三个塔各输出一个向量，融合，送进上层网络，进行推理 → 给n个物品进行打分
